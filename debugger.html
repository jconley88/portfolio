Ruby and Ruby on Rails - Time Traveling Debugger

You should easily be able to find out exactly how your application functions for every entry point; libraries and all. Unfortunately, when a single rails request can easily be 10s or even 100s of thousands of method calls, current debugging techniques make it nearly impossibly to quickly gain a high level understanding of code execution especially when delving into external libraries. This debugger fills the tooling gap by providing a high level, navigable view of all method calls and variable values. It greatly reduces the need for the normal iterative debug cycle where you add a puts or debug statement, rerun, check logs or step through, then modify and repeat.
